# Duck Rescue — мини-игра на A-Frame + AR.js

## Что здесь есть

- **start.html** — стартовая страница с QR-кодами и ссылками.
- **index.html** — основная 3D-игра на A-Frame (ПК).
- **ar.html** — AR через **камеру и гироскоп**: утки и лисы появляются вокруг точки старта. Поворачивайте телефон, чтобы осматривать сцену, и тапайте по уткам.
- **ar-gps.html** — упрощённый пример AR по GPS (несколько фиксированных точек в коде).
- **assets/** — текстуры и место для `duck_collectible.glb` и `fox_hazard.glb`.

## Сюжет игры

Игрок помогает спасти 6 уток на лесной полянке. Не подпускайте лис к уткам и не потеряйте таймер.

В игре используются:

- `a-entity`
- `a-sky` (в index.html)
- текстуры и изображения
- 3D модель из Blender (`duck_collectible.glb` и `fox_hazard.glb`) с fallback из примитивов
- текст в интерфейсе и в сцене
- свет, камера
- анимация и анимация по событию
- интерактивность (клик/тап)
- физика (в index.html — aframe-physics; в AR — логика через raycaster)
- трансформации предков и потомков

## Запуск

Запускайте через сервер (не `file://`):

```bash
cd game
npx serve .
# или
python3 -m http.server 8000
```

Откройте в браузере, например `http://localhost:3000/start.html`.

**Переход в AR:** со стартовой страницы (start.html) нажмите **«Открыть AR здесь»** или отсканируйте QR-код для AR — откроется **ar.html**. Игра стартует только после всех разрешений, таймер в AR — 2 минуты.

## AR на телефоне (ar.html)

В **ar.html** используется **камера телефона + поворот устройства**:

1. Страница запрашивает доступ к **камере**, **микрофону**, **геолокации** и **датчикам**.
2. После запуска AR-сцены утки и лисы размещаются вокруг стартовой точки обзора.
3. Поворачивайте телефон, чтобы осматривать сцену. На iPhone это наиболее совместимый вариант браузерного AR без `WebXR`.
4. Тап по утке — спасение; тап по лисе — поражение. Спасти 6 уток — победа.
5. Блок `GPS` внизу нужен только для отображения координат и статуса геопозиции; сами объекты в `ar.html` не привязаны к GPS.

Для камеры, гироскопа и GPS на телефоне обычно нужен **HTTPS** — проще всего использовать GitHub Pages.

## Демо в вузе (GitHub Pages и без интернета)

### 1. GitHub Pages (если есть Wi‑Fi)

1. Залить репозиторий на GitHub.
2. **Settings → Pages → Source**: branch `main`, папка root.
3. Открыть на телефоне: `https://<логин>.github.io/<репо>/game/ar.html`.
4. Разрешить камеру, микрофон, геолокацию и датчики — утки и лисы появятся вокруг точки старта.

### 2. Локальный сервер без интернета

Для `ar.html` на телефоне лучше использовать **HTTPS**.

Варианты:

1. Использовать **GitHub Pages**.
2. Поднять **HTTPS-туннель** (`ngrok`, `cloudflared`) к локальному серверу.
3. Для простого GPS-демо открыть отдельный `ar-gps.html`.

## Управление

- **index.html (ПК):** WASD — движение, мышь — обзор, клик по утке — спасение. Не задевать лис и не выходить за платформу.
- **ar.html (телефон):** разрешить камеру, микрофон, геолокацию и датчики. Объекты появятся вокруг точки старта. Тап по утке — спасти, тап по лисе — поражение. Счётчик и экраны победы/поражения — поверх камеры.

## Подключение модели из Blender

1. В Blender: `File → Export → glTF 2.0`, формат **glTF Binary (.glb)**.
2. Сохранить как `duck_collectible.glb` в папку `game/assets/`.
3. В игре и в AR автоматически подгрузится эта модель; если файла нет — используется fallback из примитивов A-Frame.

## ar-gps.html

Упрощённый пример: объекты привязаны к фиксированным координатам в коде. Подставьте свои latitude/longitude рядом с вашим местоположением.

## Почему модели могут не отрисовываться (AR)

Частые причины и что проверить:

1. **Гироскоп или датчики не дали доступ**  
   На iPhone и части Android-браузеров без разрешения на датчики сцена не будет реагировать на поворот телефона. Разрешите доступ и перезагрузите страницу.

2. **Страница открыта не по HTTPS**  
   Камера, GPS и датчики часто блокируются по `file://` и по обычному `http://` на телефоне. Используйте GitHub Pages или HTTPS-туннель.

3. **Разрешения не выданы полностью**  
   Если не разрешить камеру, микрофон, геолокацию или датчики, игра не стартует. Проверьте настройки сайта в браузере.

4. **Телефон ещё не включил датчики ориентации**  
   После старта сцены дайте телефону 1–3 секунды и медленно повернитесь вокруг себя, чтобы обзор стабилизировался.

5. **Файл модели (GLB) не загружается**  
   Утки по умолчанию — примитивы A-Frame (цилиндры, сферы). Если не видно даже их, дело не в GLB. Если видно примитивы, но не модель из Blender — проверьте, что `assets/duck_collectible.glb` есть и открывается по тому же адресу (тот же хост и протокол), без CORS-ошибок во вкладке «Сеть».

6. **WebGL или GPU**  
   На некоторых устройствах WebGL отключён или глючит. Проверьте в браузере, что WebGL включён; попробуйте другой телефон или браузер.

**Проверка:** откройте консоль (F12 на ПК; для телефона — удалённая отладка Chrome/Safari). Смотрите ошибки (красные) и вкладку «Сеть» — загружается ли `duck_collectible.glb` и нет ли 404/CORS.

## Если что-то не работает

- **Камера или геолокация не работают** — открывайте страницу по **HTTPS** и проверьте разрешения сайта в браузере.
- **В AR ничего не видно** — см. раздел «Почему модели могут не отрисовываться» выше. Плюс: разрешите все доступы и медленно поверните телефон 2–3 секунды после старта.
- **С телефона не открывается** — при локальном сервере телефон и ноутбук в одной сети; в адресе использовать IP ноутбука, не localhost.
